# برق نيوز v14

## التعديلات
- مفتاح OpenAI يُقرأ حسب الأولوية:
  1. Streamlit Secrets
  2. متغيرات البيئة
  3. القيم اليدوية في `config.py`
- فشل AI لا يظهر كتقييم صفر، بل `غير متاح`.
- خطأ 401 يظهر كرسالة واضحة عن المفتاح.
- تقييم AI غير المتاح لا يدخل في المتوسط أو القرار.
- تصنيف أخبار الاستحواذ حسب المرحلة:
  - مقترح غير نهائي
  - قيد المراجعة
  - موافق عليه
  - اتفاق نهائي
  - مكتمل
  - ملغى أو مرفوض
- إضافة حالة الحدث، نوع التنبيه، ودرجة ثقة النظام.
- العرض المقترح لا يُصنف كفرصة مؤكدة.
- إشعارات تيليجرام وبطاقات الموقع تعرض الحالة والثقة.
- حفظ هذه البيانات داخل سجل التعلم.

## إضافة Secrets في Streamlit
```toml
MASSIVE_API_KEY = "..."
TELEGRAM_BOT_TOKEN = "..."
TELEGRAM_CHAT_ID = "..."
OPENAI_API_KEY = "..."
```

بعد حفظ Secrets استخدم `Reboot app`.


## v14.2
- إصلاح ظهور رأي النظام كـ«غير متاح».
- إعادة تحليل الأخبار القديمة المحفوظة تلقائيًا من العنوان والملخص.

- إصلاح سجل التعلّم القديم الذي لا يحتوي رأي النظام.


## v14.3 — بدون إدخال يدوي للمفاتيح
- تمت إزالة جميع متغيرات `*_MANUAL`.
- المفاتيح تُقرأ فقط من:
  1. Streamlit Secrets
  2. متغيرات البيئة
- لا تضع المفاتيح داخل `config.py`.
- للتشغيل المحلي يمكنك إنشاء `.streamlit/secrets.toml` اعتمادًا على الملف `secrets.toml.example`.


## v14.4 — مفاتيح مباشرة داخل config.py
- تمت إعادة تصميم المفاتيح المباشر:
  - `MASSIVE_API_KEY`
  - `TELEGRAM_BOT_TOKEN`
  - `TELEGRAM_CHAT_ID`
  - `OPENAI_API_KEY`
- لا يتم استخدام Streamlit Secrets أو متغيرات البيئة.
